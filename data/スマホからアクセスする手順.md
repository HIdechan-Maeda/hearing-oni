# スマホから hearing-oni にアクセスする手順

## 基本（利用者・本番URL）

授業や配布資料で案内された **インターネット上のURL**（例: Vercel で公開された `https://xxxx.vercel.app`）を、スマホのブラウザで開きます。

- **自宅・学外・学内どこからでも**、ネットに繋がれば利用できます（授業用PCと同じWi‑Fiである必要はありません）。
- **推奨ブラウザ**: iPhone/iPad は **Safari**、Android は **Chrome**。
- アプリストアから「ダウンロード」するのではなく、**ブラウザでページを開く**形です。ホーム画面に追加する手順は `利用者向け_初回登録とホーム画面追加.md` を参照してください。

---

## 開発・動作確認用（ローカル開発）

以下は **開発者がPCでサーバーを動かしているとき** に、スマホで画面を確認するための方法です。一般の利用者向けの手順ではありません。

### 方法A: 同じ Wi‑Fi（LAN）でアクセス

1. **PC でサーバーを起動**
   ```bash
   npm run dev:lan
   ```
2. **ターミナルに表示された Network の URL を確認**
   - 例: `Network: http://192.168.11.7:3000`
   - この **192.168.x.x** が PC の IP です
3. **スマホを同じ Wi‑Fi に接続**し、ブラウザで `http://192.168.x.x:3000` を開く（ポートは表示に合わせる）

**つながらない場合の確認**

- **Mac のファイアウォール**: システム設定 → ネットワーク → ファイアウォール → オプションで「Node」または「node」の受信を許可
- **PC の IP 確認**: システム設定 → Wi‑Fi → 詳細 → TCP/IP の「IP アドレス」
- スマホが **同じ Wi‑Fi**（ゲストネットではない）か確認

---

### 方法B: トンネルでアクセス（別ネットでも可）

同じ Wi‑Fi でもつながらない場合、**ngrok** や **localtunnel** で一時的な URL を発行できます。

#### localtunnel（インストール不要で試せる）

1. PC で `npm run dev` または `npm run dev:lan` を起動したままにする（例: ポート 3000）
2. **別のターミナル**で:
   ```bash
   npx localtunnel --port 3000
   ```
3. 表示された URL（例: `https://xxx.loca.lt`）をスマホのブラウザで開く
4. 初回だけ「Click to Continue」などのボタンを押すことがあります

#### ngrok（登録が必要だが安定）

1. https://ngrok.com でアカウント作成
2. インストール: `brew install ngrok` または https://ngrok.com/download
3. PC で `npm run dev` を起動したまま、別ターミナルで:
   ```bash
   ngrok http 3000
   ```
4. 表示された `https://xxxx.ngrok.io` をスマホで開く
